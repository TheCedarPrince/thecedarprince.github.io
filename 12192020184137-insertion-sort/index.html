Technical Definition

Insertion Sort: Efficient in-place sorting algorithm for a small input. Takes as an input an array A[1 . . n] containing a sequence of length n that is to be sorted. Upon algorithm termination, the input array A is sorted. [1]

Discussion

Plain-Word Explanation

Imagine playing a card game where each card in the game are numbers 1 - 10. I find it useful when playing card games to keep my cards in order from the smallest number on a card to the biggest number. I draw 6 cards and, in my hand, they are in the order of [5,2,4,6,1,3][5, 2, 4, 6, 1, 3][5,2,4,6,1,3] - ugh! These are not in order!

I instantly set to organizing my hand by first switching 555 and 222. I then see that I can switch 444 and 555. I don't need to switch 666 as it is bigger than 555 - great! However, when I come to 111, I know that 111 is the smallest number in the game and I can instantly put it in the beginning and shift all the other cards in my hand. Finally, I come to 333 - drat, this is a pain - I have to scan the previous cards in my hand before 333 and see that it goes after 222.

Hooray! I now have a sorted hand of [1,2,3,4,5,6][1, 2, 3, 4, 5, 6][1,2,3,4,5,6] and am ready to win the game! What I just did here, is called insertion sorting!

Code Implementation

function insertionsort(input)

    @inbounds for curr = 2:length(input)
        prior = curr - 1
        while prior != 0
            if input[curr] < input[prior]
                input[curr], input[prior] = input[prior], input[curr]
            end
            curr = prior
            prior -= 1
        end
    end
    return input
end
